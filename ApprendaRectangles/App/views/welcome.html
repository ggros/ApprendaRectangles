Problem Description:

You are required to write code in the language of your choice implementing certain algorithms that analyze rectangles and features that exist among rectangles. Your implementation is required to cover the following:

  1. Intersection: You must be able to determine whether two rectangles have one or more intersecting lines and produce a result identifying the points of intersection. For your convenience, the scenario is diagrammed in Appendix 1.
  2. Containment: You must be able to determine whether a rectangle is wholly contained within another rectangle. For your convenience, the scenario is diagrammed in Appendix 2.
  3. Adjacency: Implement the ability to detect whether two rectangles are adjacent. Adjacency is defined as the sharing of a side. Side sharing may be proper or sub-line, where a sub-line share is one where one side of rectangle A is a line that exists as a set of points wholly contained on some other side of rectangle B. For your convenience, the scenario is diagrammed in Appendix 3.

Your implementation should capture the rectangle entity as well as the operations listed above. Feel free to expand on this problem as you wish. Document any expansion and provide it as part of your submission.

Your submitted source code must compile (if necessary) and the resulting executable must run on the Windows platform. Please document any library or framework dependencies.

Solution Description

I decided to use this exercice to showcase that one can grasp new concepts quickly. As such, this website was built using some bleeding edge technology that I only had seen in blog posts: Typescript, SPA, HTML Canvas

The IDE used is Visual Studio 2012 with ASP.Net MVC 4 SPA (Single Page Application) Template

The sub template used is based on DurandalJS MVVM Framework (durandaljs.com) but viewmodels have been created with TypeScript.
TypeScript definitions files are coming from https://github.com/borisyankov/DefinitelyTyped

There are actually quite a lot of choice when it comes to choosing a JS Framework as you can see here: http://addyosmani.github.com/todomvc/
I picked DurandalJS mainly because it is based on KnockoutJs which is supported by Microsoft (and inspired by XAML) and also because it was one of the template readily available for ASP.net SPA.

Required Softs

  • Visual Studio 2012 with Nuget
  • ASP.net and Web Tools 2012.2 (link)
  • TypeScript 0.8.3 For Visual Studio (Addin) - typescriptlang.org
  • Optional: Web Essentials 2012 - for enhanced Typescript support in VS(link)

Code Organization

  • Since it is an SPA Application, all the presentation Tier is in Javascript, in the App/ folder. the ASP.net MVC Framework is only used to render the layout.
  • All js is loaded using requirejs, so ViewModels are compiled by TypeScript as AMD Modules and then are loaded on demand
  • Calculations (Intersection, Containement and Adjacency) have been implemented in Javascript. (entities/shapes.ts)
  • In this version, to simplify the algorithms and data entry, we assume that rectangles are axis-aligned rectangles as shown in the Appendix
  • The page "Simple Entry" is a view that will let a user manually test the calculations.
    It is using an HTML5 Canvas and the library easeljs (from Adobe) to draw shapes on it.
    The shapes are displayed using MVVM and required to build a new Binding Handler
  • The page "Test Runner" will run all the Javascript/Typescript Tests using tsUnit
  • The template already had bootstrap.css included and design is as such the default design
